Changes to My Environment

Oct 28th, 2011

Been making some changes to my local development environment. Some of the bigger ones follow:

Switched to zsh for my shell. Still not quite sure I am going to stick to it, but giving it a chance. Quite a change from my basic bash setup I have been using for years. I had to update a few of my aliases, but it gave me a reason to cleanup my profile files. Installed Oh My ZSH! and this has been helpful. Of course I had to turn off the case insensitive expansion (that was annoying), but it is making the environment a lot nicer.

Reinstalled DTerm, and this is a nice addition for a lot of the aliases and one-liners I like to run while I am working.

Big rework of my local GeekTool desktop. The release of GeekTool in the Mac App Store caused me to finally reinstall this. I’ll post my scripts to GitHub one of these days.

Started this here GitHub Pages blog, and there is an About Me section here. I am still working on the layout and such, so it is a work in progress.

Using MacVim more and more, but I am still quite tied to Textmate. Is there a developers editor that implements the newer features available in Lion? My biggest problem is I have internalized a lot of the snippets, and not having those is a drag.

Cleaned up the old home dir, so that most files are now nicely organized into folders. It was getting pretty messy.

Upgraded my local Tomcat/Solr install so I can start working on my bookmarking app again. Had to do a full ant clean/ant compile/ant dist cycle because Solr was complaining about missing methods. Started using Pow again for local testing, but I haven’t made up my mind on that one. I have also been playing with JQuery mobile for my little note-taking app.

Some other things I have been looking at include Scientific Linux, Riak, MongoDB/GridFS, MCollective, and this Nagios/Mod-Gearman thing. I want to look at that a lot more soon.
